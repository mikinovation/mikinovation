## とあるフロントエンジニアの開発日記

- 動画の趣旨
  - 自分が監視された状態で、より集中力の高い状態で開発を行いたい
  - フロントエンド領域の有益な情報を発信したい
  - 他のフロントエンドエンジニアとコミュニケーションを取りながら開発したい
- 企画(ルール)
  - 長期的な目標を立てない。その代わりにその日できることを全力でやる
  - 毎日投稿はしない。基本はやりたいと思った時にやる
  - 1日の作業時間は2時間から3時間程度
  - 作業配信をしたときに次やるべきことを決めてissueを立てる
  - いっぺんにたくさんissueを作らない
- 企画(内容)
  - 技術トピック
  - 気になった記事の紹介
  - その日の作業配信
